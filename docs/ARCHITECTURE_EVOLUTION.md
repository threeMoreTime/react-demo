# 架构演进与历史决策 (Architecture Evolution & Decisions)

本文件记录了 `React Quest Laboratory` 从雏形到 1.0 旗舰版过程中经历的重大技术决策 (Plan Summary)，旨在为后续开发者提供“为什么这么做”的上下文审计。

## 历史策略压缩 (Context Compression)

### 1. 题库模块化拆分 (The Batching Strategy)
- **背景**: 题库规模从 5 题激增至 100 题，单一 `quests.ts` 文件行数过千，导致 IDE 语法高亮卡顿且 Git 冲突风险极高。
- **决策**: 实施 **分片导入机制**。将题目按功能维度（基础、Hook、并发等）拆分为 `batch_0` 至 `batch_10` 的独立文件库。
- **影响**: 实现了题库的按需维护与极速定位。

### 2. 难度阶梯标准化 (Level 1-10 Definition)
- **背景**: 早期难度定义（如 Easy/Hard）过于主观，无法体现 React 学习曲线的陡峭程度。
- **决策**: 引入 **10 级阵法体系**。
  - `Level 1-4`: 基础 JSX 与 闭包陷阱。
  - `Level 5-8`: 高级组件模式与生态（React Query/Headless）。
  - `Level 9-10`: 源码级并发模型（Lane）、宏任务调度与 RSC。
- **影响**: 为用户提供了清晰的成长路径。

### 3. 星图导航范式转移 (Star Map Transition)
- **背景**: 传统的 `Steps` (步骤条) 在展示 100 级关卡时会溢出屏幕并引发严重渲染性能问题。
- **决策**: 放弃线性导航，采用 **“星图大航海”全息地图 (LevelMapModal)**。利用 CSS Grid 响应式布局搭配批次 Tabs 切换。
- **影响**: 成功承载了海量题目的索引，并赋予了项目“游戏化闯关”的独特视觉体验。

### 4. Vue 对比教学闭环 (Pedagogy Loop)
- **背景**: 纯 React 教学文档往往让 Vue 转型开发者感到迷惑。
- **决策**: 强制所有题库数据必须包含 `vueAnalogy` 对象，形成 `功能对标 -> 哲学差异 -> 底层深度解析 (Deep Dive)` 的闭环。
- **影响**: 确立了本项目的核心市场竞争力。

---

## 遗留决策 (Legacy Notes)
- **渲染引擎**: 坚持使用 `pre` 标签配合占位符正则切分 (`split('_______')`) 而不引入重型 Web 编辑器，以保持实验室的“残卷感”与运行能效。
- **状态模型**: 采用 `localStorage` 进行极简持久化，暂不引入后端数据库。
